MatchZoo

@Xieydd · Jan 23, 2018 · 2 min read

[MatchZoo]

MatchZoo是文本匹配工具包，它的发展重点是促进设计、比较和共享深度文本匹配模型。有例如DRMM、MatchPyramid、MV_LSTM、aNMM、DUET、ARC-Ⅰ、ARC-Ⅱ、DSSM以及CDSSM，通过统一接口设计的深度匹配模型。与MatchZoo相关的潜在任务包括文档检索、问题回答、会话响应排序、意译识别等。我们总是很高兴收到来自我们所有的MatchZoo用户的任何代码整合、建议和评论。

任务 文本1 文本2 对象
意义识别 string1 string2 分类
文本含义 文本 意义 分类
QA 问题 回答 分类和得分
会话 对话框 回应 分类和得分
信息检索 询问 文档 得分

安装

$ git clone https://github.com/faneshion/MatchZoo.git
$ cd MatchZoo
$ sudo python setup.py install

试运行例子:

$ python matchzoo/main.py --phase train --model_file examples/toy_example/config/arci_ranking.config
$ python matchzoo/main.py --phase predict --model_file examples/toy_example/config/arci_ranking.config

综述

[图片-MatchZoo框架 —来自于网络]

工具包由三部分组成: - 数据准备 - 模型结构 - 训练和评价

三者被设计为数据流的pipeline

数据准备

数据准备模块的目的是将不同文本匹配任务的数据集转换成一个统一的格式，作为深度匹配模型的输入。用户提供数据集，其中包含对文本的成对和标签，模块生成以下文件。

  • 字典：将每个单词的映射记录到一个名为wid的惟一标识符。过于频繁的单词(例如:stopwords)，太过稀少或太吵(例如，传真号码)可以被预先定义的规则过滤掉。

  • 语料库文件：将每个文本映射到一个名为tid的唯一标识符，以及该文本中包含的单词标识符序列。注意，每个文本被截断或填充到用户定制的固定长度。

  • 关系文件：用于存储两个文本之间的关系，每一行包含一对tids和相应的标签。

  • 详细的输入数据格式：输入数据格式的详细的解释可以在MatchZoo/data/example/readme.md找到

    • corpus.txt：每一行对应一个文档。第一列是文档ID，接下来的单词是在标记化之后的文档。
    • corpus_preprocessed.txt：每一行对应一个文档。第一列是文档id，第二列是文档长度，后面是文档中的单词id。

    • relation_train.txt/relation_valid.txt/relation_test.txt：每一行都是“label query_id doc_id”，可以用来进行文档检索、通道检索、回答语句选择等。对于每个查询，文档都是按标签排序的。这些标签可以是二进制的，也可以是多级的。

    • sample.txt:每一行都是文档的原始查询和原始文档文本。格式为“标签\t查询\t document_txt”。

    • word_dict.txt：这个词词典。每一行都是单词和word_id。

    • corpus_preprocessed_dssm.txt/word_dict_dssm.txt：这些文件的格式与 “corpus_preprocessed.txt/word_dict.txt” 相同。由于DSSM使用了基于三字母的字典，所以我们为DSSM模型创建了单词字典的分离文件。

对于分类，大多数文件的格式与排名的情况是一样的。唯一的区别是 “relation_train.txt/relation_valid.txt/relation_test.txt”的格式。每一行都是“label document_id1 document_id2”。以释义为例，我们想要预测两句话是否具有同样的含义，即使用深度文本匹配模型。

示例输入数据可以在MatchZoo/data/example.你可以将原始的sample.txt转换成这些文件通过

MatchZoo/examples/test_preparation_for_classify.py 实现分类，或者MatchZoo/examples/test_preparation_for_ranking.py 实现评分

建模

使用Keras库很方便的一层一层建立深度匹配模型。Keras 库提供了一组广泛应用于神经模型的常用层，如卷积层、池化层、dense层。为了进一步促进深文本匹配模型的构建，我们扩展了Keras库，为文本匹配专门设计了一些层接口。

此外，该工具箱还实现了两种具有代表性的深度文本匹配模型，即以表示为中心的模型和以交互为中心的模型Guo et al.。

训练和评估

为了学习深度匹配的模型，该工具包提供了各种用于回归、分类和排序的目标函数。例如，与ranking相关的目标函数包括几个众所周知的pointwise, pairwise 和 listwise losses。在训练阶段，用户可以灵活地选择不同的目标函数进行优化。一旦一个模型被训练，这个工具箱就可以用来得到一个匹配的分数，预测一个匹配的标签，或者将目标文本(例如，一个文档)与输入文本相对应。

定标结果

这里我们采用WikiQA数据集，配合MatchZoo的使用。WikiQA是一个流行的基准数据集，用于回答问题。我们提供一个脚本来下载数据集，比匹配MatchZoo的数据格式。在模型目录中，有许多关于WikiQA数据集的模型的配置。

以DRMM为例。可以这样训练

$ python matchzoo/main.py --phase train --model_file examples/wikiqa/config/drmm_wikiqa.config

测试

$ python matchzoo/main.py --phase predict --model_file examples/wikiqa/config/drmm_wikiqa.config

可以对不同的模型进行，benchMark测试。
