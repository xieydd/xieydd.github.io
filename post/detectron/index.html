Detectron

@Xieydd · Jan 25, 2018 · 5 min read

[Detectron]

Detectron是FAIR(Facebook AI Research)实现最先进的目标检测算法的软件，包含Mask R-CNN.使用Python以及Caffe2的深度学习框架支持

在FAIR，Detectron已经开启多个研究项目，包括Feature Pyramid Networks for Object Detection、 Mask R-CNN、Detecting and Recognizing Human-Object Interactions、Focal Loss for Dense Object Detection、Non-local Neural Networks, Learning to Segment Every Thing、Data Distillation: Towards Omni-Supervised Learning,即特征金字塔网络用于对象检测、Mask R-CNN、检测和识别人类动作交流、密集对象检测的焦点损失、非本地神经网络、学会分割每一件事、数据精炼:面向全监督学习

[Example Mask R-CNN output]

1.介绍

Deterctron的目标是提供一个高质量，高性能的目标检测代码库。它的设计是灵活的，快速实现和评估论文。Detection包含以下目标检测算法的实现:

  • Mask R-CNN – marr Prize at ICCV 2017
  • RetinaNet – Best Student Paper Award at ICCV 2017
  • Faster R-CNN
  • RPN
  • Fast R-CNN
  • R-FCN

使用以下主干神经网络框架:

  • ResNeXt{50,101,152}
  • ResNet{50,101,152}
  • Feature Pyramid Networks(with ResNet/ResNeXt)
  • VGG16

额外的主干框架很容易实现，具体参考

设置教程和Jupyter Server

如果在云计算机上运行这一切，默认情况下，您可能不会有一个UI或方法来查看IPython notebooks。通常，您将在本地使用ipython notebook启动它们，您将看到一个localhost:8888网页弹出的是笔记本运行的目录。下面的示例将向您展示如何启动Jupyter server并通过SSH通道连接到远程。

首先配置您的云服务器以接受端口8889，或者您想要的任何东西，但是要在以下命令中更改端口。在AWS上，您可以通过向服务器的安全组添加一个规则来实现这一目标，允许在端口8889上输入TCP。否则你会调整iptables。

#启动Jupyter Server
$ jupyter notebook --no-browser --port=8889

然后创建SSH通道。这将把云服务器的Jupyter实例传递到您的本地主机8888端口，供您在本地使用。下面的例子是在您如何连接AWS(您的公共证书)之后进行模板化的。your-public-cert.pem是您自己的公共证书，ubuntu@superrad-gpu-instance.compute -1.amazonaws.com是您的登录到您的云服务器。您可以在AWS上轻松地获取这一信息，并在ssh之后复制该部分，并在下面的命令中进行交换。

$ ssh -N -f -L localhost:8888:localhost:8889 -i "your-public-cert.pem" ubuntu@super-rad-GPU-instance.compute-1.amazonaws.com

我们提供Dockerfile，可以使用它创建Detectron镜像.如果你是要预训练Caffe2镜像(例如从docker镜像库)

FROM caffe2:cuda8-cudnn6-all-options

# Install the COCO API
RUN git clone https://github.com/cocodataset/cocoapi.git
WORKDIR /cocoapi/PythonAPI
RUN make install

# Clone the Detectron repository
RUN git clone https://github.com/facebookresearch/detectron /detectron

# Set up Python modules
WORKDIR /detectron/lib
RUN make

# Build custom ops
RUN make ops

# Go to Detectron root
WORKDIR /detectron

排错

如果出现Caffe2安装问题，请阅读相关Caffe2安装指导的故障排除部分,我们为Caffe2和Detectron提供了额外的故障排除提示

Caffe2运算符分析

Caffe2附带性能分析支持，您可能会发现对您的操作人员进行基准测试或调试非常有用,例如BatchPermutationOp test。分析支持不是默认生成的，您可以通过在运行Caffe2 CMake时设置-DUSE_PROF = ON标志来启用它

CMake无法找到CUDA和cuDNN

有时候，CMake在你的机器上很难找到CUDA和cuDNN文件

在构建Caffe2时，你可以将CMake指向CUDA和cuDNN文件通过:

$ cmake .. \
  # insert your Caffe2 CMake flags here
  -DCUDA_TOOLKIT_ROOT_DIR=/path/to/cuda/toolkit/dir \
  -DCUDNN_ROOT_DIR=/path/to/cudnn/root/dir

类似地，在构建定制的检测操作时，你可以使用:

$ cd $DETECTRON/lib
$ mkdir -p build && cd build
$ cmake .. \
  -DCUDA_TOOLKIT_ROOT_DIR=/path/to/cuda/toolkit/dir \
  -DCUDNN_ROOT_DIR=/path/to/cudnn/root/dir
$ make

请注意，您可以使用相同的命令来让CMake使用在您的机器上安装的多个版本的CUDA和cuDNN的特定版本。

Protobuf错误

Caffe2使用protobuf作为序列化格式，需要3.2.0或更新版本。如果您的protobuf版本较旧，您可以从Caffe2 protobuf子模块构建protobuf，然后使用该版本.

构建Caffe2 protobuf子模块

# CAFFE2=/path/to/caffe2
$ cd $CAFFE2/third_party/protobuf/cmake
$ mkdir -p build && cd build
$ cmake .. \
  -DCMAKE_INSTALL_PREFIX=$HOME/c2_tp_protobuf \
  -Dprotobuf_BUILD_TESTS=OFF \
  -DCMAKE_CXX_FLAGS="-fPIC"
$ make install

将Caffe的的CMake指向新建成的protobuf:

$ cmake .. \
  # insert your Caffe2 CMake flags here
  -DPROTOBUF_PROTOC_EXECUTABLE=$HOME/c2_tp_protobuf/bin/protoc \
  -DPROTOBUF_INCLUDE_DIR=$HOME/c2_tp_protobuf/include \
  -DPROTOBUF_LIBRARY=$HOME/c2_tp_protobuf/lib64/libprotobuf.a

如果您同时安装了系统和anaconda包，那么您可能也会遇到protobuf的问题。这可能会导致问题，因为版本可能在编译时或运行时混合。通过遵循上面的命令，也可以克服这个问题。

Caffe2 Python二进制文件

如果您遇到CMake无法找到所需的Python路径的问题(例如在virtualenv中)，您可以尝试将Caffe2的CMake指向Python库，并使用：

$ cmake .. \
  # insert your Caffe2 CMake flags here
  -DPYTHON_LIBRARY=$(python2 -c "from distutils import sysconfig; print(sysconfig.get_python_lib())") \
  -DPYTHON_INCLUDE_DIR=$(python2 -c "from distutils import sysconfig; print(sysconfig.get_python_inc())")
Caffe2 NNPACK构建

Detectron不需要用NNPACK支持构建的Caffe2。如果您在Caffe2的安装过程中遇到了NNPACK相关的问题，您可以通过设置-dusennpack=OFF cflag标志来安全地禁用NNPACK。

Caffe2 OpenCV构建

类似于上面的NNPACK情况，您可以通过设置-DUSE_OPENCV = OFF CMake标志来禁用OpenCV

COCO API未定义符号错误

如果您遇到了由于未定义的符号而导致的COCO API导入错误，如报告所指出的那样,确保你的Python版本没有混淆。例如，如果您同时安装了系统和conda numpy，可能会出现这个问题

CMake找不到Caffe2的时候

如果您遇到了CMake在构建自定义操作符时无法找到Caffe2包的问题，请确保您已经将make install作为您的Caffe2安装过程的一部分

基本成功

Detectron会自动的下载模型通过指定的URL –wts 参数.这个工具会可视化出检结果以PDF的形式到特定的文件夹 –output-dir.

注意:

在对自己的高分辨率图像进行推理时,Mask R-CNN可能会很慢,因为花费大量时间将预测的掩码上采样到原始图像分辨率（这尚未被优化).你可以诊断这个issue如果tools/infer_simple.py对misc_mask的时间报告很(超过20-90ms).解决方法是首先调整图像的大小，使短边大约在600-800像素之间（确切的选择无关紧要），然后在调整大小后的图像上运行推理。

使用Detectron训练一个模型

这是一个小教程，展示如何在COCO上训练模型,该模型将是一个使用ResNet-50-FPN为基础的的端到端训练的Faster R-CNN.为了本教程的目的，我们将使用较短的训练时间表和较小的输入图像大小，以便训练和推理将相对较快.因此，COCO Boxs AP相对于我们来说相对baselines较低,这个例子仅用于指导目的（即不用于与出版物比较)

  • 使用一GPU训练
python2 tools/train_net.py \
    --cfg configs/getting_started/tutorial_1gpu_e2e_faster_rcnn_R-50-FPN.yaml \
    OUTPUT_DIR /tmp/detectron-output

期望的结果:

1.输出(模型，验证集合检测)将会被保存到/tmp/detectron-output

2.在Maxwell一代GPU（例如M40）上，训练应花费大约4.2小时

3.推断时间应该是每个图像80ms(也在M40上）

4.在coco_2014_minival上Box AP应该是在22.1%左右(超过3次的标准差应该是在0.1%)

  • 多GPU训练

我们还提供配置来说明使用2,4和8个GPU的训练，使用的学习时间表将近似等于上面的1个GPU使用的时间表.配置在configs/gettingstarted/tutorial{2,4,8}gpu_e2e_faster_rcnn_R-50-FPN.yaml.如果训练一个2GPU的任务:

python2 tools/train_net.py \
    --multi-gpu-testing \
    --cfg configs/getting_started/tutorial_2gpu_e2e_faster_rcnn_R-50-FPN.yaml \
    OUTPUT_DIR /tmp/detectron-output

注意我们同时加上了–multi-gpu-testing的参数指示Detectron在多个GPU上进行并行推理（在本例中为2;参见配置文件中的NUM_GPUS），训练结束后.

期望的结果:

1.培训需要大约2.3小时（2 x M40）

2.推断时间应该是每个图像80ms(在并行的两个M40上,为时间的一半）

3.在coco_2014_minival上Box AP应该是在22.1%左右(超过3次的标准差应该是在0.1%)

要了解如何调整学习计划（“线性缩放规则”）,请学习这些教程配置文件并阅读我们的论文 Accurate, Large Minibatch SGD: Training ImageNet in 1 Hour.注意不像我们的这个教程,我们所有的设置都是基于8GPU的,如果少于8个或者想改变minibatch的,您必须了解如何根据线性缩放规则来操作训练计划。

注意:

  • 这个例子使用相对较低的GPU计算模型因为Caffe2 Python操作系统的开销相对较高.结果，随着GPU数量从2个增加到8个，缩放相对较差（例如，使用8个GPU的训练花费大约0.9个小时，仅比使用1个GPU快4.5倍）。随着越来越多，使用更多的GPU计算重型模型，缩放比例也随之提高
